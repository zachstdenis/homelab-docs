# Operations Documentation

This folder contains operational documentation for running, maintaining, recovering, and improving the homelab over time.

These files focus less on what the environment is, and more on how it is operated in practice.

## Contents

- [Runbooks](runbooks.md) - short operational procedures for common recovery and maintenance tasks
- [Maintenance and Updates](maintenance-and-updates.md) - update strategy, validation steps, and maintenance priorities
- [Backups and Recovery](backups-and-recovery.md) - recovery priorities, backup posture, and known resilience gaps
- [Roadmap](roadmap.md) - near-term, medium-term, and longer-term priorities for the homelab

## Purpose

The documents in this folder are meant to answer questions like:

- how should this environment be maintained?
- what should be checked after changes or outages?
- what are the current recovery priorities?
- what risks or weak points are known?
- what is the next work that actually matters?

## Notes

This folder is intentionally operations-focused.

Architecture and reference material live in the core documentation folder, while incident writeups and service deployment writeups live in their own sections so those topics do not clutter the day-to-day operational docs.
